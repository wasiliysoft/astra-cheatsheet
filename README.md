# 🛡️ Шпаргалка системного администратора Astra Linux (Смоленск)
*Справочник по управлению пользователями, безопасности, мониторингу и специфическим механизмам защиты*

---

## 👤 Управление пользователями и контроль входа

- **Блокировка пароля** — `sudo usermod -L <user>`
- **Разблокировка пароля** — `sudo usermod -U <user>`
- **Запрет входа (shell)** — `sudo usermod -s /sbin/nologin <user>`
- **Принудительный выход из системы** — `sudo pkill -u <user>`
- **Просмотр активных сессий** — `who` или `w`
- **История входов** — `last`
- **Неудачные попытки входа** — `lastb`

### Сброс пароля
1. Перезагрузите компьютер
2. Нажмите Esc при загрузке
3. В меню выберите Astra Linux → нажмите `E`
4. Найдите строку `linux /boot/vmlinuz...`
5. В конец строки добавьте: `init=/bin/bash`
6. Нажмите `Ctrl+X`
7. Введите команды:
`mount -o remount,rw /`
`passwd <user>`
8. Введите новый пароль 2 раза
9. Перезагрузитесь: `reboot -f`
10. Рекомендуется сменить пароль ещё раз уже в работающей системе

## 🔐 Парольная политика, блокировки и сроки

### Жизненный цикл пароля (`chage`)
- Просмотр текущих лимитов — `sudo chage -l <user>`
- Макс. срок действия пароля (90 дней) — `sudo chage -M 90 <user>`
- Мин. интервал между сменами (7 дней) — `sudo chage -m 7 <user>`
- Предупреждение за 14 дней — `sudo chage -W 14 <user>`
- Дата истечения аккаунта — `sudo chage -E 2025-12-31 <user>`

### Сложность пароля
- Файл настройки — `/etc/pam.d/common-password`
- Параметры — `minlen`, `ucredit`, `dcredit`, `lcredit`, `ocredit`

### Блокировка при неудачных входах
- Просмотр числа неудачных попыток — `sudo pam_tally2 --user <user>`
- Сброс счетчика — `sudo pam_tally2 --user <user> --reset`
- Файл настройки — `/etc/pam.d/common-auth` (параметры: `deny=5`, `unlock_time=900`)

### Ограничения доступа
- Разрешение/запрет входа по IP, группам, терминалам — `/etc/security/access.conf`
- Блокировка входа всем, кроме root — создать файл `/etc/nologin`
- Авторазрыв сессии Bash при бездействии (300 сек) — добавить `TMOUT=300` в `/etc/profile`

## 🛡️ Специфика Astra Linux (МРД, мандатный контроль, целостность)

### Мандатные метки
- Показать мандатный контекст текущей сессии — `pdp-id`
- Просмотр мандатных меток файла/папки — `pdp-ls -l <file>`
- Установить иерархический уровень (0–63) — `sudo pdp-setlvl <level> <file>`
- Установить неиерархическую категорию — `sudo pdp-setcat <category> <file>`
- Просмотр процессов с их мандатными метками — `ps -eZ`

### Контроль целостности
- Статус контроля целостности исполняемых файлов — `astra-mic-control status`
- Включение контроля — `astra-mic-control enable`
- Проверка целостности установленных пакетов — `dpkg -V`
- Проверка ФС по базе хешей (если установлен AIDE) — `sudo aide --check`

### Межсетевой экран
- Просмотр правил — `sudo ufw status verbose`
- Включение/выключение — `sudo ufw enable` / `sudo ufw disable`

## 🛠 Системный мониторинг, логи и диагностика

### Ресурсы и процессы
- Загрузка CPU/RAM в реальном времени — `top` или `htop`
- Использование ОЗУ в мегабайтах — `free -m`
- Свободное место на дисках — `df -h`
- Размер папок в текущей директории — `du -sh *`
- Поиск процесса по имени — `ps aux | grep <name>`
- Принудительное завершение процесса — `kill -9 <PID>`

### Логи и аудит
- Мониторинг авторизации (SSH, sudo) — `tail -f /var/log/auth.log`
- Системный лог в реальном времени — `tail -f /var/log/syslog`
- Поиск ошибок входа — `grep "Failed password" /var/log/auth.log`
- Все команды через sudo — `journalctl _COMM=sudo`
- Логи конкретной службы (live) — `journalctl -u <service> -f`

### Сеть и порты
- IP-адреса и интерфейсы — `ip a`
- Проверка доступности — `ping <host>`
- Открытые порты и процессы (замена `netstat`) — `ss -tunlp`
- Трассировка маршрута — `traceroute <host>`
- HTTP-заголовки ответа — `curl -I <url>`
- Перехват трафика (сниффер) — `tcpdump -i eth0`
- Сканирование портов и версий служб — `nmap -sV localhost`

## 🔍 Поиск уязвимостей и аномалий

- Файлы с SUID-битом (риск повышения привилегий) — `find / -perm -4000`
- Файлы, доступные для записи всем (world-writable) — `find / -perm -2 -type f`
- Поиск скрытых файлов — `find / -name ".*" -ls`
- Просмотр атрибутов файлов (например, неизменяемый флаг `i`) — `lsattr`

## 📦 Управление пакетами

- Обновление списка пакетов — `sudo apt update`
- Обновление установленных программ — `sudo apt upgrade`
- Установка — `sudo apt install <package>`
- Полное удаление с конфигами — `sudo apt purge <package>`

## ⚙️ Управление службами (Systemd)

- Запуск/остановка/перезапуск — `systemctl start/stop/restart <service>`
- Состояние службы — `systemctl status <service>`
- Добавление/удаление из автозагрузки — `systemctl enable/disable <service>`

## 📁 Работа с файлами и ACL (расширенные права)

- Просмотр ACL — `getfacl <file>`
- Установка ACL — `setfacl -m u:<user>:rwx <file>`
- Удаление ACL — `setfacl -x u:<user> <file>`
- Копирование с сохранением прав — `cp -rp <source> <dest>`
- Рекурсивная смена владельца — `sudo chown -R user:group <dir>`

---

## 💾 Резервное копирование и восстановление

- Бэкап директории (tar) — `tar -czf backup.tar.gz /path/to/dir`
- Распаковка — `tar -xzf backup.tar.gz`
- Бэкап прав доступа — `getfacl -R /path > permissions.acl`
- Восстановление прав — `setfacl --restore=permissions.acl`
- Клонирование диска (dd) — `sudo dd if=/dev/sda of=/dev/sdb bs=64K`

---

## 🔄 Загрузка и восстановление системы

- Обновление GRUB — `sudo update-grub`
- Восстановление GRUB — `sudo grub-install /dev/sda`
- Перезагрузка — `sudo reboot`
- Выключение — `sudo shutdown -h now`
- Сброс забытого пароля root — через однопользовательский режим: `init=/bin/bash`
- Проверка файловой системы — `sudo fsck /dev/sda1`

## 👥 Группы и права sudo

- Добавление пользователя в группу — `sudo usermod -aG <group> <user>`
- Удаление из группы — `sudo deluser <user> <group>`
- Просмотр групп пользователя — `groups <user>`
- Редактирование sudoers — `sudo visudo`
- Проверка sudo-прав — `sudo -l`

## 📊 Мониторинг дисков и I/O

- I/O статистика дисков — `iostat -x 1`
- Активность диска в реальном времени — `iotop`
- SMART-статус диска — `sudo smartctl -a /dev/sda`
- Поиск больших файлов — `find / -type f -size +100M -exec ls -lh {} \;`
- Иноды (мелкие файлы) — `df -i`

## 🌐 Работа с репозиториями Astra

- Просмотр подключенных репозиториев — `cat /etc/apt/sources.list`
- Добавление репозитория — `sudo add-apt-repository <repo>`
- Очистка кэша APT — `sudo apt clean && sudo apt autoclean`
- Удаление неиспользуемых зависимостей — `sudo apt autoremove`

- Загруженные модули ядра — `lsmod`

- Выгрузка модуля — `sudo rmmod <module>`

- Информация о процессоре — `lscpu`
- Информация о USB — `lsusb`
- Информация о PCI — `lspci`
