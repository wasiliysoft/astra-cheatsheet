# 🛡️ Шпаргалка системного администратора Astra Linux (Смоленск)

Единый справочник по управлению пользователями, безопасности, мониторингу и специфическим механизмам защиты Astra Linux.

---

## 👤 Управление пользователями и контроль входа

| Действие | Команда |
| :--- | :--- |
| **Блокировка пароля** | `sudo usermod -L <user>` |
| **Разблокировка пароля** | `sudo usermod -U <user>` |
| **Запрет входа (shell)** | `sudo usermod -s /sbin/nologin <user>` |
| **Принудительный выход из системы** | `sudo pkill -u <user>` |
| **Просмотр активных сессий** | `who` или `w` |
| **История входов** | `last` |
| **Неудачные попытки входа** | `lastb` |

## 🔐 Парольная политика, блокировки и сроки

### Жизненный цикл пароля (`chage`)
- `sudo chage -l <user>` — просмотр текущих лимитов
- `sudo chage -M 90 <user>` — макс. срок действия пароля (90 дней)
- `sudo chage -m 7 <user>` — мин. интервал между сменами (7 дней)
- `sudo chage -W 14 <user>` — предупреждение за 14 дней
- `sudo chage -E 2025-12-31 <user>` — дата истечения аккаунта

### Сложность пароля
Настраивается в `/etc/pam.d/common-password` (параметры `minlen`, `ucredit`, `dcredit`, `lcredit`, `ocredit`).

### Блокировка при неудачных входах
- `sudo pam_tally2 --user <user>` — просмотр числа неудачных попыток
- `sudo pam_tally2 --user <user> --reset` — сброс счетчика
- Настройка в `/etc/pam.d/common-auth`: `deny=5`, `unlock_time=900`

### Ограничения доступа
- `/etc/security/access.conf` — разрешение/запрет входа по IP, группам, терминалам
- `/etc/nologin` — создание файла блокирует вход всем, кроме root
- `TMOUT=300` — авторазрыв сессии Bash при бездействии (добавить в `/etc/profile`)

## 🛡️ Специфика Astra Linux (МРД, мандатный контроль, целостность)

### Мандатные метки
- `pdp-id` — показать мандатный контекст текущей сессии
- `pdp-ls -l <file>` — просмотр мандатных меток файла/папки
- `sudo pdp-setlvl <level> <file>` — установить иерархический уровень (0–63)
- `sudo pdp-setcat <category> <file>` — установить неиерархическую категорию
- `ps -eZ` — просмотр процессов с их мандатными метками

### Контроль целостности
- `astra-mic-control status` — статус контроля целостности исполняемых файлов
- `astra-mic-control enable` — включение контроля
- `dpkg -V` — проверка целостности установленных пакетов (изменялись ли файлы)
- `sudo aide --check` — проверка ФС по базе хешей (если установлен AIDE)

### Межсетевой экран
- `sudo ufw status verbose` — просмотр правил
- `sudo ufw enable/disable` — включение/выключение

## 🛠 Системный мониторинг, логи и диагностика

### Ресурсы и процессы
- `top` или `htop` — загрузка CPU/RAM в реальном времени
- `free -m` — использование ОЗУ в мегабайтах
- `df -h` — свободное место на дисках
- `du -sh *` — размер папок в текущей директории
- `ps aux | grep <name>` — поиск процесса по имени
- `kill -9 <PID>` — принудительное завершение процесса

### Логи и аудит
- `tail -f /var/log/auth.log` — мониторинг авторизации (SSH, sudo)
- `tail -f /var/log/syslog` — системный лог в реальном времени
- `grep "Failed password" /var/log/auth.log` — поиск ошибок входа
- `journalctl _COMM=sudo` — все команды через sudo
- `journalctl -u <service> -f` — логи конкретной службы (live)

### Сеть и порты
- `ip a` — IP-адреса и интерфейсы
- `ping <host>` — проверка доступности
- `ss -tunlp` — открытые порты и процессы (замена `netstat`)
- `traceroute <host>` — трассировка маршрута
- `curl -I <url>` — HTTP-заголовки ответа
- `tcpdump -i eth0` — перехват трафика (сниффер)
- `nmap -sV localhost` — сканирование портов и версий служб

## 🔍 Поиск уязвимостей и аномалий

- `find / -perm -4000` — файлы с SUID-битом (риск повышения привилегий)
- `find / -perm -2 -type f` — файлы, доступные для записи всем (world-writable)
- `find / -name ".*" -ls` — поиск скрытых файлов
- `lsattr` — просмотр атрибутов файлов (например, неизменяемый флаг `i`)

## 📦 Управление пакетами

- `sudo apt update` — обновление списка пакетов
- `sudo apt upgrade` — обновление установленных программ
- `sudo apt install <package>` — установка
- `sudo apt purge <package>` — полное удаление с конфигами

---

## ⚙️ Управление службами (Systemd)

- `systemctl start/stop/restart <service>` — запуск/остановка/перезапуск
- `systemctl status <service>` — состояние службы
- `systemctl enable/disable <service>` — добавление/удаление из автозагрузки
